3Co jednotlivé metriky znamenají a jak se počítají

Všechny metriky pocházejí z oficiálního NESTFUL scoreru. Rozsah 0–1 (vyšší = lepší).

F1 Func

F1 přes množinu názvů funkcí (korpusové macro)

Měří, zda model volá správné funkce (jen názvy, bez pořadí a argumentů). Počítá se jako macro-F1 přes ~900 tříd (názvů funkcí) pomocí MultiLabelBinarizer + f1_score. Macro = průměr přes třídy, ne přes vzorky — proto u řídkého slovníku vychází vysoko (viz výhrady).

F1 Param

F1 přes dvojice (funkce, jméno_parametru)

Jako F1 Func, ale třídy jsou páry funkce + parametr. Měří, zda model používá správné argumenty správných funkcí. Přísnější než F1 Func.

Partial Sequence Accuracy

% pozic, kde (název + argumenty) sedí s goldem

Po groundingu proměnných (reference $var2.result$ se přepíše na producenta) a zarovnání délek se porovná každý krok sekvence. Partial = podíl správných pozic. Mnohem těžší než F1, protože vyžaduje i správné pořadí a hodnoty.

Full Sequence Accuracy

1, pokud sedí CELÁ sekvence; jinak 0 (průměr přes úlohy)

Nejpřísnější strukturní metrika: celá predikovaná sekvence callů se musí přesně shodovat s gold sekvencí (po groundingu). U ReAct je často 0, protože model dává konkrétní hodnoty místo $varN$ referencí.

Win Rate (execution-based)

% úloh, kde predikované cally jdou SPUSTIT a dají správnou gold odpověď

Jediná metrika založená na reálné exekuci: scorer spustí predikované funkce (s timeoutem přes signal.SIGALRM — proto jen Linux) a porovná finální výsledek s gold odpovědí (s tolerancí na zaokrouhlení). Toleruje alternativní cesty — model může dojít ke správné odpovědi jinou (validní) sekvencí, proto bývá Win > Full Accuracy.

4Curriculum trénink — progrese

Pilotní curriculum (malý eval per stupeň). Stupně postupují k delším sekvencím (1-call → 4-call+). strict_gold_trace_pass = přesná reprodukce gold trace; off_f1_func = oficiální re-score.

strict_gold_trace_pass padá s obtížností: stage 1 ≈ 0.38 → stage 3 ≈ 0.01 → stage 4 = 0.00. Model se delší multi-call sekvence v pilotu (16 úloh/epocha) nikdy nenaučil reprodukovat. Reálné zlepšení ukázala až větší results_v2 varianta (+3.5 pp executor accuracy).

5Výhrady k interpretaci

„F1 ≈ 90 %" neznamená 90 % úspěšnost. F1 Func je korpusové macro-F1 přes ~904 názvů funkcí, kde 881 z 904 tříd je singleton/near-singleton (≤2 výskyty) a 84 % volání jsou jen add/subtract/multiply/divide. Macro průměruje třídy stejně → pojmenování (často zjevně nabízené) funkce táhne metriku k ~0.92. Pro kontext uvádíme i micro-F1 (~0.84) a per-task set-match (~0.31) / seq-match (~0.28). Skutečnou úspěšnost úlohy ukazují Full Acc / Win Rate (sekundárně Partial).
ReAct vs Direct nejsou přímo srovnatelné s paperem. Paper Table 1 je Direct Prompting. ReAct emituje konkrétní hodnoty místo $varN$ referencí → Full ≈ 0, nižší Partial. Pro paper-srovnatelná čísla používej naše Direct řádky.
Metodika je jednotná. Všechny běhy přepočítané stejným oficiálním NESTFUL scorerem ze surových predikcí/trajektorií — žádné míchání definic. Win Rate pro 4 hlavní běhy přepočten na Linuxu.
